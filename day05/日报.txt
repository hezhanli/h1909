16.java 中的集合有哪些？
java 中的集合分为，单列集合Collection,双列集合Map。
Collection 分为：List 和 Set。
List 接口的特点：元素可重复，有序。
list 接口的实现类：
ArrayList:底层实现是数组，查询快，增删慢，线程不安全，效率高； Vector:底层实现是数组，查询快，增删慢，线程安全，效率低；
LinkedList:底层实现是链表，增删快，查询慢，线程不安全，效率高； Set 接口的特点：元素不可重复，无序。
Set 接口实现类：
HashSet:底层实现 hashMap，数组+链表实现，不允许重复，无序。TreeSet:底层实现红黑二叉树，有序
Map 接口的特点：key-value 键值对形式
Map 接口实现类：
HashMap：底层数组+链表，线程不安全效率高； TreeMap：底层红黑二叉树实现，可实现排序；


17.HashMap 底层原理
1、HashMap 底层是数组+链表实现, 默认初始化容量为16，也就是说数组索引值为 0-15，
每个数组中存储一个链表。
2、当空间使用达到 0.75 后，会对数组进行扩容，每次扩容翻倍。
3、存储元素时，存储对象为Entry对象；entry 对象包含四个信息，key、value、hash 值、链表地址值；存储时，会根据key的hash 值%(模以)16 计算，得到数组中存储的索引位置。
4、但是有可能发生 hash 碰撞现象，即key不相同，却有一样的 hash 值，这样的话，就以链表的形式进行存储，第一个 entry 存在链表顶端，再有 hash 值一致的 entry 存入，则链接在第一个元素之后。
5、put()方法存储元素时，先根据key计算出hash值，然后用这个hash值确定在数组中的位置，再把value值放进去，如果这个位置本来没放东西，就会直接放进去，如果之前就有，就会生成一个链表，把新放入的值放在头部


18.hashMap 和 hashTable 的区别？
相同点：
1、二者都是 key-value 的双列集合；底层都是通过数组+链表方式实现的
不同点：
1、线程安全性不同
Hashtable 中的方法是线程安全的，而 HashMap 中的方法是非
线程安全的。在多线程并发的环境下，可以直接使用 Hashtable，但使用 HashMap时就必须要自己增加同步处理。
2、hashMap 允许 null 键和 null 值，但是 hashtable 不允许。
4、HashMap 是开发中常用的类，实现线程安可以用 ConcurrentHashMap(读音:康克润特哈希map)。

19.什么是线程安全？如何保证线程安全？
线程安全，首先要明白线程的工作原理。
jvm 有一个主存，而每个线程有自己的工作内存，一个线程对一个变量进行操作时，都要在自己的工作内存里面建立一个拷贝，操作完之后再写入主内存。多个线程同时操作同一个变量，就可能会出现不可预知的结果。
而用synchronized 的关键是建立一个监视器（监视器可以是要修改的变量，也可以是对象或方法），然后通过给监视器加锁来实现线程安全，每个线程在获得这个锁之后，要执行完
(1) 从主存复制变量到当前工作内存
(2) 执行代码，改变共享变量值
(3) 用工作内存数据刷新主存相关内容 
过程，才会释放它得到的锁。这样就实现了所谓的线程安全。
线程安全就是说多线程访问同一代码，不会产生不确定的结果。编写线程安全的代码依靠线程同步。
